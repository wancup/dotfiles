# CLAUDE.md

- 日本語では**必ず和文ベタ組み**を基本とし、日本語と英数字の間に不要な空白を入れない
- 一時ファイルを作成する際はプロジェクト内の`.__tmp/`ディレクトリを使用する

## コマンド

- `rm`を使用する際は許可プロンプトを省略するため`-f`を指定する

## 実装

- 後方互換性を過剰に意識せず、全体設計としてどうあるべきかを重視する

## Git

- worktreeを作成するときは、リポジトリルートでfishの`git_worktree_add`コマンドを使う
- commitメッセージはなるべく一文で書く
- commitメッセージには**なぜこの変更をおこなったか**というWhyを含める
- commitメッセージの形式は直近のログを参照して慣習に合わせる
