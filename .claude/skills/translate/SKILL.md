# 翻訳スキル（日本語 ⇔ 英語）

## 入力

`$ARGUMENTS`

## 手順

1. **入力の解釈**
   - ファイルパスが渡された場合: Readツールでファイル内容を読み取り、その全文を翻訳対象とする
   - URLが渡された場合: WebFetchで内容を取得し、その全文を翻訳対象とする
   - それ以外: 渡されたテキストをそのまま翻訳対象とする

2. **言語の判定**
   - 翻訳対象が日本語なら英語に翻訳する
   - 翻訳対象が英語なら日本語に翻訳する

3. **翻訳の出力**
   - 必ず **3パターン** の翻訳を提示する
   - 各パターンには以下のラベルを付ける:
     - **パターン1（自然・流暢）**: 最も自然で読みやすい訳
     - **パターン2（正確・忠実）**: 原文の構造や意味に忠実な訳
     - **パターン3（簡潔・カジュアル）**: 短くシンプルな表現の訳
   - 各パターンの後に、そのニュアンスや特徴を **原文の言語** で簡潔に説明する

4. **コミットメッセージ向け翻訳**
   - 3パターンの後に、**コミットメッセージ向け** の翻訳案を1つ追加で提示する
   - 簡潔で動詞始まりの英語表現にする

## ルール

- 技術用語、コード識別子、固有名詞はそのまま維持する
- ファイルやURLの内容を翻訳する場合も、全文に対して3パターンすべてを出力する
- 翻訳結果のみを出力し、手順の説明や前置きは不要
